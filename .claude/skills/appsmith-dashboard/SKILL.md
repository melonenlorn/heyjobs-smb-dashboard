# Appsmith Dashboard — HeyJobs SMB Steering Cockpit

## Architecture Overview

```
JS_Config   → static config: teams, levels, thresholds, quarter helpers
JS_Data     → all KPI calculations, reads via _r(queryName)
JS_Init     → page load: Q_Users_Team, buildTeamsFromQuery, parallel queries, auto-snapshot
JS_Filters  → filter state (storeValue), teamOptions, quarterOptions, applyFilters
JS_Snapshots→ git-committed raw query snapshots (generated by update_quarter.py)
Q_*         → Salesforce SOQL queries (metadata.json per query)
```

## Critical Appsmith Constraints

### No ES6 Getters in JSObjects
```js
// ❌ BREAKS silently — getter returns undefined at runtime
get TEAMS() { return { wolves: {...} } }

// ✅ Use direct mutable properties
TEAMS: { wolves: {...} },
```

### No Dynamic SOQL in Salesforce SAAS Plugin
The Appsmith Salesforce SAAS plugin does NOT evaluate `{{}}` template bindings inside `formData`. SOQL must be **hardcoded** in `metadata.json`. Dynamic values (rep IDs, dates) are updated by `scripts/update_quarter.py`.

### Reactive Refresh Pattern
Async query completion alone does NOT trigger `defaultModel` re-evaluation. Always end `initDashboard()` with:
```js
await storeValue('_refreshTs', Date.now());
```
And include `appsmith.store._refreshTs` in every Custom Widget's `defaultModel` binding.

### No `window`, no `eval`, no `import`
JSObjects are sandboxed. Reference other JSObjects by name directly (`JS_Config.TEAMS`, not `window.JS_Config`).

### Custom Widget Events
Events fire via `appsmith.triggerEvent('eventName')`. The `evtMap` inside the widget HTML must match the exact event key names used in Appsmith bindings. Keys are typically `wolves/titans/locos`, NOT old manager names.

### Rep-Discovery Query (role-based, no hardcoded IDs)
```sql
SELECT Id, Name, ManagerId, Manager.Name, UserRole.Name
FROM User
WHERE IsActive = true
  AND UserRole.Name LIKE 'Inside Sales IC%'
ORDER BY Manager.Name, Name
```
→ `Manager.Name` liefert die Team-Zuordnung. Keine hardcodierten IDs nötig.

### ⚠ IsActive = false Trap
Promoted/renamed SF accounts become `IsActive = false`. Always use `WHERE IsActive = true`. Old accounts (e.g. `0059L000000JKFGQA4` = "Philipp xBahls") sind tot — **nie** in MANAGERS config verwenden. Den neuen Account per SOQL suchen.

| Opportunity | `Winback_Pilot__c` | > 0 = Pilot-Deal |
| Opportunity | `RecordType.Name` | Filter: `!= 'Customer Self Service'` for rep-driven |
| Task | `TaskSubtype = 'Call'` OR `CallType IN ('Inbound','Outbound')` | Dials |

### buildTeamsFromQuery() — Manager.Name Matching
```js
buildTeamsFromQuery() {
  const mgrs = JS_Config.MANAGERS[q] || ...;
  const mgrNameToKey = {};
  for (const [key, m] of Object.entries(mgrs)) {
    mgrNameToKey[m.tlName] = key;
  }
  for (const r of records) {
    if (JS_Config.EXCLUDED_REPS.indexOf(r.Name) !== -1) continue;
    const teamKey = mgrNameToKey[r.Manager ? r.Manager.Name : null];
    if (teamKey) { /* IC zuordnen */ }
  }
}
```

## JS_Data Patterns

### _r(queryName) — Dualmode Live/Snapshot
```js
_r(queryName) {
  const aq = JS_Config.getActiveQuarter();
  if (!aq.isCurrent) {
    // 1. Git snapshot (update_quarter.py)
    const snap = JS_Snapshots.data[aq.label];
    if (snap?.queries?.[queryName]) return snap.queries[queryName];
    // 2. Appsmith Store snapshot (auto-saved on page load)
    const storeSnap = appsmith.store['snap_' + aq.label.replace(' ', '_')];
    if (storeSnap?.queries?.[queryName]) return storeSnap.queries[queryName];
    return [];
  }
  // Live: liveMap[queryName].data.output.records
}
```
**Immer `JS_Data._r('Q_XYZ')` statt direktem Query-Zugriff verwenden.**

### Dynamic Query Names (z.B. CW/PW)
```js
const sfx = period === 'CW' ? '_CW' : '_PW';
JS_Data._r('Q_Calls' + sfx).forEach(...)
// ❌ NICHT: const q = period === 'CW' ? Q_Calls_CW : Q_Calls_PW; JS_Data._r(q)
```

## JS_Init Patterns

### Page Load Flow
```js
async initDashboard() {
  // 1. Team-Struktur aus SF
  await Q_Users_Team.run();
  JS_Config.buildTeamsFromQuery();

  // 2. Historisches Quartal → Daten aus Snapshot, keine Queries
  if (!JS_Config.getActiveQuarter().isCurrent) {
    await storeValue('_refreshTs', Date.now());
    return;
  }

  // 3. Alle Live-Queries parallel
  await Promise.allSettled([ ...alle 29 Queries... ]);

  // 4. Auto-Snapshot speichern (überschreibt vorherigen — immer aktuell)
  await JS_Init.saveSnapshot();

  // 5. Reactive refresh
  await storeValue('_refreshTs', Date.now());
}
```

### Auto-Snapshot (vollautomatisch, kein Button)
`saveSnapshot()` speichert alle Query-Rohdaten in `appsmith.store['snap_Q1_2026']` (persistent). Wird nach jedem Init aufgerufen → beim letzten Dashboard-Öffnen im Quartal ist der Snapshot aktuell.

## Quartalswechsel-Workflow

```bash
python3 scripts/update_quarter.py 2 2026
```
1. Aktualisiert Datumsbereiche in QTD-Queries
2. Ermittelt aktuelle Rep-IDs via `sf data query` (UserRole-basiert)
3. Aktualisiert `OwnerId IN (...)` in allen Queries
4. `git commit + push`

Snapshot kommt automatisch von Appsmith (kein SF-CLI in Script nötig).

**Beim Quartalswechsel manuell prüfen:**
- `JS_Config.MANAGERS` für neues Quartal vorhanden?
- `JS_Config.REP_LEVELS` für neue Reps ergänzen?
- `EXCLUDED_REPS` aktuell?

## metadata.json Struktur (Queries)

```json
{
  "unpublishedAction": {
    "actionConfiguration": {
      "formData": {
        "CUSTOM_ACTION": {
          "params": [{"key": "q", "value": "SELECT ... FROM ..."}],
          "path": "query/?q=SELECT ..."
        },
        "command": "CUSTOM_ACTION"
      }
    }
  }
}
```
Beide Felder (`params[0].value` UND `path`) müssen immer synchron aktualisiert werden.

## Häufige Bugs & Fixes

| Bug | Ursache | Fix |
|-----|---------|-----|
| `.map is not a function` | ES6 Getter in JSObject | Getter → direkte Property |
| Dashboard lädt, aber zeigt keine Daten | `defaultModel` evaluiert nicht neu | `storeValue('_refreshTs', Date.now())` + `_refreshTs` in `defaultModel` |
| Team fehlt komplett | Manager `IsActive = false` | Neuen aktiven SF-Account in MANAGERS eintragen |
| Custom Widget-Button reagiert nicht | `evtMap` keys passen nicht zu `triggerEvent()` keys | evtMap keys = teamKeys (`wolves/titans/locos`) |
| 400 Bad Request bei SF Query | `{{}}` Bindings in SAAS Plugin nicht evaluiert | SOQL hardcoden, update_quarter.py für Updates |
| Historisches Quartal zeigt keine Daten | Snapshot existiert nicht | Dashboard während altem Quartal geöffnet? Snapshot per `storeValue` prüfen |

## Neue KPI hinzufügen — Checkliste

1. **Query erstellen**: `pages/Dashboard/queries/Q_NeueKPI/metadata.json`
2. **JS_Init**: Query zu `Promise.allSettled([...])` hinzufügen
3. **JS_Init.saveSnapshot()**: Query zu `queries: { ... }` hinzufügen
4. **JS_Data._r() liveMap**: Query-Referenz ergänzen
5. **JS_Data**: Neue `*ByRep()` Funktion mit `JS_Data._r('Q_NeueKPI')`
6. **JS_Data.allRepKPIs()**: Neues Feld in Rep-Objekt eintragen
7. **Widget**: Binding auf `JS_Data.heroKPIs()` oder `defaultModel.repRows`
8. **defaultModel**: Neue Query in Dependency-Array aufnehmen
